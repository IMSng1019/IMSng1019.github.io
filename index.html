服务器介绍

版本平台

JAVA Fabric 1.20.4

服务器特色

Java 与基岩互通，Java 端不限正版／非正版，基岩端需微软登录，都需要通过专属 QQ 群验证。

有LuckPerms权限管理，全玩家下发tp指令，限制op管理权限，无VIP特权。内置所需模组与优化。不换周目，永不删档。不限生电/大型机器，欢迎各类玩家加入。

硬件配置

Intel Core i7-14700K + 10 GB RAM，提供稳定流畅的游戏体验

玩法定位

  • 生存向
  • 生电向
  • 建筑向
  • 指令向（不会发op，限制较大详细规制见条目四）

关于MC服务器

本服务器已经开服一年多了，经历了由单人存档到radmin到frp到简幻欢再到云服务器的过程。在不断的优化和增加功能后，我决定将服务器对外公开。在漫长的时间道路上，感谢各位玩家的一路陪伴。

核心玩法

1. 生存探索

服务器开启了1.21预览实验性玩法，可以体验1.21生存冒险新内容。服务器开启了死亡不掉落且下发tp指令，无后顾之忧。

2. 生存电路

服务器不限制生电，你可以自由建造你所想建造的机器（无功能的卡服机除外）。

3. 建筑设计

自由地建造你所想建造的建筑。服务器的方块小镇MOD可以使你的建筑更加美观，服务器内的生电机器可以让你的建筑过程更加便捷。

管理团队

由腐竹与资深玩家（两位生电、一位指令）组成，腐竹会及时处理举报、技术疑难与新手问题。

规则及告知

禁止破坏

在不影响整体美观及功能下你可以自由建筑。未经同意不得破坏及修改他人建筑。禁止乱拆生电机器。不要乱修改其中任意一个部件。优化必须经过相应作者允许。

友善交流

严禁人身攻击、种族歧视、政治宗教话题；对话中请保持尊重与耐心。

防骗措施

任何官方渠道均不会让你转账、泄露密码；服务器没有VIP，SVIP，除管理以外的op等一切特权。遇到请在群内先行核实。

指令玩家和OP

目前服务器内已开放的指令已经足够普通玩家使用，故不会给op给普通玩家。如果你想在服务器内建造指令小游戏请联系腐竹并提供相应地图。如果你有代码/技术或解决现有问题的能力且有意愿改善服务器，请联系腐竹。

技术与性能保障

1. 性能优化

  • 核心优化：优化mod由腐竹整理保证发挥电脑最好的性能与兼容性支持。
  • 插件精简：仅挑选必要的防护、监控与优化插件。客户端无需安装。
  • 必要模组下载：
    1. 使用服务器内优化一键包：
      Automodpack
    2. 自己配置：
      architectury-11.1.17-fabric.jar
      yuushya-modelling-1.20.4-fabric-2.2.0.jar
      yuushya-1.20.4-fabric-2.2.2.jar

2. 数据备份与安全

  • 自动存档：不定期手动备份。
  • DDoS 防护：独立ipv4 带有普防 有备用ip
  • 权限系统：使用 LuckPerms 细分权限组，明确权限，无VIP组存在 仅有op与非op只区分 op均为管理团队成员 权限有限制
  • QQ群验证：每一位玩家都可被及时追溯。

常见问题

为什么要加 QQ 验证？

原本是没有的，但因为加了盗版密码登录，为了防止"熊"玩家乱刷屏／破坏 / 炸服并提供一个稳定的追溯路径，加强社区管理；同时可在群内及时发布公告，所以加了QQ群验证。

服务器为什么选择1.20.4？

①1.20.4相比于之前版本有足够的mod生态 ②服务器内有依靠nbt的指令小游戏 因高版本nbt改变 升至1.20.5及以上版本会导致现有nbt指令不可用③1.20.4可以体验1.21的预览性更新

后面会考虑1.21吗？

如果能解决nbt的问题就升

服务器内的已有的op是怎么来的？

现服务器内的op（加上腐竹一共3位）来自最开始radmin时期的服务器的部分原始股且具有一定特长。

怎么成为op？

不下发op，此点见条目四

automodpack是必装的吗？

不是必装的，你可以在安装必需mod后自己配置。

服务器会删区块吗？

一般不会，除非区块内有无法处理的卡服器和禁人塔
